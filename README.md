# 2. Diseño  
La arquitectura **cliente-servidor** se basa en dividir el sistema en dos partes:

- **Cliente:** el navegador del usuario (HTML, CSS, JS).  
  Aquí se muestran las páginas y se captura la interacción.

- **Servidor:** (en proyectos reales) sería la parte encargada de procesar datos, responder peticiones y guardar información.

En esta práctica solo usamos **cliente**, aunque el concepto sigue siendo el mismo.

# 3. Despliegue  
Hemos usado **GitHub Pages** para publicar la aplicación.  
Pasos realizados:

1. Crear repositorio en GitHub  
2. Subir `index.html` y `fav.ico`  
3. Activar GitHub Pages desde Settings → Pages  
4. Elegir la rama **main** y la carpeta **root**  
5. Comprobar la URL generada
